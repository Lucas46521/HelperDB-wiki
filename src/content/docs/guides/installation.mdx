# Installation

Install Helper.db using npm:

```bash
npm install helper.db
```

Or using yarn:

```bash
yarn add helper.db
```

Or using pnpm:

```bash
pnpm add helper.db
```

## Prerequisites

Before installing Helper.db, make sure you have:

- **Node.js** version 14.0 or higher
- **npm** or **yarn** package manager
- A Node.js project initialized with `package.json`

## TypeScript Support

Helper.db includes built-in TypeScript definitions. No additional `@types` package needed!

```typescript
import { Database, Query, Backup } from 'helper.db';

const db: Database = new Database('myapp.db');
```

## Environment Setup

### Development Environment

For development, you might want to use file-based databases:

```javascript
const { Database } = require('helper.db');
const db = new Database('./dev-database.db');
```

### Production Environment

For production, consider using environment variables:

```javascript
const { Database } = require('helper.db');
const dbPath = process.env.DATABASE_PATH || './production.db';
const db = new Database(dbPath);
```

### Testing Environment

For testing, use in-memory databases:

```javascript
const { Database } = require('helper.db');
const db = new Database(':memory:'); // Temporary, in-memory database
```

## Common Installation Issues

### Node.js Version Compatibility

If you encounter compatibility issues, ensure you're using Node.js 14 or higher:

```bash
node --version
```

### Permission Issues

On Unix systems, you might need to adjust file permissions:

```bash
chmod 755 ./database-directory
```

### SQLite Dependencies

Helper.db uses SQLite under the hood. If you encounter native module compilation issues:

```bash
npm rebuild
# or
npm install --build-from-source
```
